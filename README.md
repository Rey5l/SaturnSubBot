# EarnMoneyBot

Telegram-бот для заработка на заданиях (Tgrassa и Flyer — подписки на каналы) и вывода средств через **Crypto Bot** (Crypto Pay API).

## Возможности

- **Grs / Fly задания** — каналы для подписки (из своей базы). Пользователь подписывается → нажимает «Проверить подписку» → бот проверяет через Telegram API и зачисляет награду на баланс.
- **Баланс** — отображение текущего баланса (зачисления за подписки и от админа).
- **Вывод в Crypto Bot** — вывод накопленных средств в @CryptoBot через Crypto Pay API.
- **Админ** — зачисление баланса (`/addbalance`), добавление каналов для заданий (`/addchannel`), список каналов (`/channels`).

## Установка

1. Клонируйте проект и перейдите в каталог:
   ```bash
   cd EarnMoneyBot
   ```

2. Создайте виртуальное окружение и установите зависимости:
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate   # Windows: .venv\Scripts\activate
   pip install -r requirements.txt
   ```

3. Создайте бота в Telegram через [@BotFather](https://t.me/BotFather) и скопируйте токен.

4. Скопируйте пример конфигурации и отредактируйте `.env`:
   ```bash
   cp .env.example .env
   ```
   Заполните:
   - `BOT_TOKEN` — токен от BotFather.
   - `CRYPTO_PAY_API_TOKEN` — токен приложения Crypto Pay (см. ниже).
   - `ADMIN_IDS` — ваш Telegram user_id (можно узнать через @userinfobot).
   - При необходимости измените `TGRASSA_BOT_LINK` и `FLYER_BOT_LINK` на актуальные ссылки.

### Настройка Crypto Pay (вывод средств)

1. Откройте [@CryptoBot](https://t.me/CryptoBot) → **Crypto Pay** → **Create App**.
2. Создайте приложение и скопируйте **API Token** в `.env` как `CRYPTO_PAY_API_TOKEN`.
3. В настройках приложения включите **Transfers** (Security → Transfers → Enable), чтобы иметь возможность отправлять средства пользователям.
4. Пополните баланс приложения (чтобы было чем платить при выводе).

Для тестов можно использовать [@CryptoTestnetBot](https://t.me/CryptoTestnetBot) и в `.env` указать `CRYPTO_PAY_NETWORK=TEST_NET`.

## Запуск

```bash
source .venv/bin/activate
python main.py
```

## Админ-команды

- `/addbalance` — зачислить баланс пользователю (далее бот запросит user_id и сумму).
- `/balance_admin <user_id>` — посмотреть баланс пользователя.
- `/addchannel` — добавить канал для заданий (платформа Grs/Fly, username канала, награда, название). Пользователи смогут подписаться и получить награду на баланс после проверки.
- `/channels` — список добавленных каналов-заданий.

Админами считаются пользователи, чей `user_id` указан в `ADMIN_IDS` в `.env`.

## Каналы для подписки и проверка

Каналы заданий хранятся в своей БД. Админ добавляет их через `/addchannel` (username канала, награда, платформа Grs или Fly).

**Важно:** чтобы бот мог проверять подписку, его нужно **добавить в канал** (как администратора с правом «просмотр участников» или просто как участника — для публичных каналов часто достаточно, чтобы бот был в канале). Иначе метод `getChatMember` не сможет проверить пользователя.

## Структура проекта

- `main.py` — точка входа, запуск бота.
- `config.py` — настройки из `.env`.
- `database.py` — SQLite: пользователи, баланс, история выводов.
- `handlers/` — обработчики команд и кнопок (меню, Tgrassa, Flyer, баланс, вывод, админ).
- `services/crypto_pay.py` — работа с Crypto Pay API (баланс, transfer).
- `data/bot.db` — база SQLite (создаётся автоматически).

## Важно

- Баланс в боте — виртуальный. Админ зачисляет его вручную после того, как пользователь выполнил задания на Tgrassa/Flyer (или по своей схеме).
- Реальный вывод идёт с баланса вашего приложения Crypto Pay. Убедитесь, что на нём достаточно средств и включены Transfers.
